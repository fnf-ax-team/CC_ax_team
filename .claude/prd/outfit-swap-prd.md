# PRD: 착장 스왑 (Outfit Swap)

> 작성일: 2026-02-13
> 상태: Draft

---

## 1. 개요

### 1.1 목적
기존 이미지의 **얼굴/포즈/배경을 100% 보존**하면서 **착장만 교체**하는 워크플로.
스튜디오 재촬영 없이 다양한 착장 조합을 빠르게 생성.

### 1.2 타겟 사용자
- 마케팅팀: 같은 모델로 여러 착장 버전 필요
- MD팀: 신상품 착장 이미지 빠른 생성
- 이커머스팀: 상세페이지용 착장 변형

### 1.3 핵심 가치
| 가치 | 설명 |
|------|------|
| 시간 절약 | 재촬영 없이 착장 교체 |
| 일관성 | 동일 모델/포즈/배경 유지 |
| 비용 절감 | 스튜디오/모델 비용 감소 |

### 1.4 워크플로 카테고리
**스왑** - 얼굴 유지, 착장 변경, 포즈 유지

- [x] MLB 브랜드 로고 정확도 (위치/색상/크기)

## 3. 입출력 정의

### 3.1 입력

| 입력 | 타입 | 필수 | 설명 |
|------|------|------|------|
| 소스 이미지 | Image | O | 얼굴/포즈/배경 보존 대상 (1장) |
| 착장 이미지 | Image[] | O | 교체할 착장 (N장, 누끼 or 착용샷) |
| 비율 | string | O | 1:1, 3:4, 4:5, 9:16 등 |
| 수량 | int | O | 1, 3, 5장 |
| 화질 | string | O | 1K, 2K, 4K |

### 3.2 출력

| 출력 | 타입 | 설명 |
|------|------|------|
| 생성 이미지 | Image[] | 착장 교체된 결과물 |
| 검증 결과 | dict | 점수, 등급, 이슈 |
| 분석 로그 | JSON | 소스/착장 분석 결과 |

### 3.3 출력 폴더
```
Fnf_studio_outputs/outfit_swap/{timestamp}/
├── result_01.png
├── result_02.png
├── result_03.png
└── analysis_log.json
```

## 4. 워크플로 설계

### 4.1 단계별 흐름

```
┌──────────────────────────────────────────────────────────────────┐
│                     착장 스왑 파이프라인                           │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. 입력 수집                                                     │
│     └─ 소스 이미지 경로                                           │
│     └─ 착장 폴더 경로                                             │
│     └─ 옵션 선택 (비율/수량/화질)                                  │
│                                                                  │
│  2. VLM 분석 (병렬)                                               │
│     ├─ analyze_source() → 얼굴/포즈/배경 추출                     │
│     └─ analyze_outfit() → 착장 디테일 추출 (기존 모듈 재사용)       │
│                                                                  │
│  3. 프롬프트 조립                                                  │
│     └─ build_outfit_swap_prompt()                                │
│         ├─ 소스 보존 지시 (얼굴/포즈/배경 EXACT)                   │
│         └─ 착장 교체 지시 (디테일/로고/색상 EXACT)                 │
│                                                                  │
│  4. 이미지 생성                                                    │
│     └─ generate_outfit_swap()                                    │
│         ├─ 프롬프트                                               │
│         ├─ 소스 이미지 (직접 전달)                                 │
│         └─ 착장 이미지 (직접 전달)                                 │
│                                                                  │
│  5. 검증                                                          │
│     └─ OutfitSwapValidator.validate()                            │
│         ├─ face_identity (소스 vs 결과)                           │
│         ├─ pose_preservation (소스 vs 결과)                       │
│         ├─ outfit_accuracy (착장 vs 결과)                         │
│         └─ background_preservation (소스 vs 결과)                 │
│                                                                  │
│  6. 재생성 (실패 시)                                               │
│     └─ 실패 원인별 프롬프트 강화 → 최대 2회                        │
│                                                                  │
│  7. 출력                                                          │
│     └─ 저장 + 결과 반환                                           │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

## 5. 검증 기준

### 5.1 검증 기준 정의

| 기준 | 비중 | 설명 | Auto-Fail |
|------|------|------|-----------|
| face_identity | 25% | 얼굴 동일성 (소스 vs 결과) | < 90 |
| pose_preservation | 25% | 포즈 유지 (관절 위치, 자세) | < 95 |
| outfit_accuracy | 35% | 착장 정확도 (색상/로고/디테일) | < 80 |
| outfit_draping | 10% | 드레이핑 자연스러움 | < 70 |
| background_preservation | 5% | 배경 유지 | < 90 |

### 5.2 등급 체계

| 등급 | 점수 | 판정 |
|------|------|------|
| S | 95+ | 바로 사용 |
| A | 90+ | 바로 사용 |
| B | 85+ | 확인 필요 |
| C | 75+ | 재생성 권장 |
| F | 75- | 재생성 필수 |

### 5.3 Auto-Fail 조건

| 조건 | 설명 |
|------|------|
| 얼굴 다른 사람 | face_identity < 90 |
| 포즈 변경됨 | pose_preservation < 95 |
| 착장 색상 불일치 | outfit color match < 80 |
| 로고 누락/변형 | logo accuracy < 85 |
| 배경 변경됨 | background_preservation < 90 |

### 5.4 MLB 브랜드 특화 검증

| 체크 항목 | 기준 |
|----------|------|
| NY/MLB 로고 위치 | front_center vs left_chest 정확 |
| 로고 색상 | 화이트/네이비/레드 정확 |
| 로고 크기 | 참조 대비 ±10% |
| 로고 타입 | embroidered/chenille/printed 구분 |

### 5.5 재생성 로직

| 실패 기준 | 프롬프트 강화 방향 |
|----------|-------------------|
| face_identity 실패 | "EXACT same face from source" 강조 |
| pose_preservation 실패 | "EXACT same pose, joint positions" 강조 |
| outfit_accuracy 실패 | 착장 디테일 반복 + "EXACT color/logo" |
| outfit_draping 실패 | "Natural fabric draping for pose" 추가 |

**재시도 설정:**
- 최대 재시도: 2회
- Temperature: 0.2 → 0.15 → 0.1

## 6. 기술 설계

### 6.1 모듈 구조

```
core/outfit_swap/
├── __init__.py           # 통합 진입점
├── analyzer.py           # 소스 이미지 분석 (얼굴/포즈/배경)
├── prompt_builder.py     # 프롬프트 조립
├── generator.py          # 생성 + 검증 루프
├── validator.py          # 검증 로직
└── templates.py          # VLM 프롬프트 템플릿
```

### 6.2 기존 모듈 재사용

| 모듈 | 용도 |
|------|------|
| `core/outfit_analyzer.py` | 착장 분석 (이미 구현됨) |
| `core/validators/base.py` | 검증기 베이스 클래스 |
| `core/api.py` | API 키 로테이션 |
| `core/config.py` | 모델 상수 |

### 6.3 API 사용

| 용도 | 모델 | Temperature |
|------|------|-------------|
| 소스 분석 | VISION_MODEL | 0.1 |
| 착장 분석 | VISION_MODEL | 0.2 |
| 이미지 생성 | IMAGE_MODEL | 0.2 |
| 검증 | VISION_MODEL | 0.1 |

### 6.4 이미지 전달 순서

```
1. 프롬프트 (텍스트) - 착장 스왑 지시 + 착장 상세 설명
2. 소스 이미지 (첫 번째) - 얼굴/포즈/배경 보존 대상
3. 착장 이미지들 - 교체할 착장 참조
```

**핵심 원칙:**
- 소스 이미지: **API 직접 전달** (얼굴/포즈/배경 정확 보존)
- 착장 이미지: **API 직접 전달 + VLM 분석 텍스트** (디테일 정확도)

## 7. 핵심 불변량 (Core Invariants)

### 7.1 보존 규칙

| 구분 | 요소 | 변경 허용 |
|------|------|----------|
| **must_preserve** | 얼굴 (동일 인물) | 절대 불가 |
| **must_preserve** | 포즈 (관절 위치) | 절대 불가 |
| **must_preserve** | 배경 | 절대 불가 |
| **must_preserve** | 체형 | 절대 불가 |
| **must_change** | 착장 | 필수 변경 |
| **flexible** | 착장 드레이핑 | 포즈에 맞게 자연스럽게 |

### 7.2 드레이핑 규칙

착장이 소스 포즈에 맞게 자연스럽게 적용되어야 함:
- 팔 구부림 → 소매 주름
- 다리 자세 → 바지 실루엣
- 몸통 각도 → 상의 드레이프
- 물리적으로 불가능한 착장 X

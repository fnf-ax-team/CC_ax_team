# 문서 동기화 (Task 완료 후 실행)

매 Task 완료 후 프로젝트 문서를 업데이트하여 다음 세션에서도 컨텍스트를 유지합니다.

## 업데이트 대상 파일

1. **CLAUDE.md** - 프로젝트 가이드, 명령어, 아키텍처
2. **DASHBOARD_ARCHITECTURE.md** - 대시보드 구조, 컴포넌트, API
3. **DATA_STRUCTURE.md** (또는 최신 버전) - 데이터 스키마, 컬럼 정의
4. **DASHBOARD_QUERIES.md** - Snowflake 쿼리, 데이터 변환 로직

## 수집할 정보

이번 세션에서 다음 내용을 파악하세요:

### 1. 코드 변경사항
```bash
git status
git diff --stat
git log --oneline -5
```

### 2. 새로 추가/수정된 내용
- 새 컴포넌트, 서비스, API 엔드포인트
- 새 타입 정의
- 설정 변경 (포트, 환경변수 등)
- 버그 수정 또는 기능 개선

### 3. 사용자가 제공한 추가 정보
- 비즈니스 로직 설명
- 데이터 구조 변경
- 특별 주의사항

## 문서 업데이트 가이드

### CLAUDE.md 업데이트 항목
- 새 명령어 추가
- 아키텍처 변경 (새 서비스, 컴포넌트)
- API 엔드포인트 추가/변경
- 환경변수 변경
- 주요 패턴 변경

### DASHBOARD_ARCHITECTURE.md 업데이트 항목
- 대시보드 섹션 순서 변경
- 컴포넌트 구조 변경
- 프론트엔드/백엔드 구조 변경
- UI/UX 가이드라인 변경
- 트러블슈팅 추가

### DATA_STRUCTURE.md 업데이트 항목
- 새 컬럼 추가
- 데이터 타입 변경
- 계산 로직 변경
- 속성값 변경 (luster, texture, length 등)

### DASHBOARD_QUERIES.md 업데이트 항목
- Snowflake 쿼리 수정
- 새 쿼리 추가
- 데이터 변환 로직 변경

## 실행 절차

1. **현재 상태 파악**
   - git 상태 확인
   - 이번 세션 대화 내용 리뷰

2. **각 문서 읽기**
   - CLAUDE.md
   - DASHBOARD_ARCHITECTURE.md
   - DATA_STRUCTURE.md (최신 버전)
   - DASHBOARD_QUERIES.md

3. **업데이트 필요 여부 판단**
   - 변경사항과 문서 내용 비교
   - 누락된 정보 식별

4. **문서 업데이트**
   - 각 파일에 새 정보 추가
   - 오래된 정보 수정
   - 날짜 업데이트 (Last Updated)

5. **DATA_STRUCTURE 버전 관리**
   - 중요한 스키마 변경 시 새 버전 파일 생성
   - 파일명: `DATA_STRUCTURE_vYYMMDD.md`

## 주의사항

- 문서가 업데이트 불필요하면 "변경 없음"으로 표시
- 기존 내용을 삭제하지 말고 추가/수정만 수행
- 한글 주석과 설명 유지
- 코드 예시는 실제 코드 기반으로 작성

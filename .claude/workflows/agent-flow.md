## 에이전트 간 통신 규약

### 통신 매트릭스

| From → To | 트리거 | 요청 형식 | 응답 형식 |
|-----------|--------|----------|----------|
| PLANNER → CRITIC | PRD 초안 완료 | "{워크플로} PRD를 검증해줘" | APPROVED/NEEDS_REVISION/REJECTED |
| CRITIC → PLANNER | 수정 필요 | "다음 사항 수정 필요: [...]" | 수정된 PRD |
| CRITIC → EXECUTOR | APPROVED | "구현 진행해도 됨" | - |
| EXECUTOR → TESTER | 구현 완료 | "테스트 실행해줘" | 테스트 결과 리포트 |
| TESTER → EXECUTOR | 버그 발견 | "다음 버그 수정 필요: [...]" | 수정된 코드 |
| TESTER → GATEKEEPER | 테스트 통과 | 테스트 결과 리포트 | - |
| GATEKEEPER → CRITIC | 릴리즈 전 | "{워크플로} 불변량 준수 확인" | 체크리스트 결과 |
| GATEKEEPER → ALL | RELEASE | "core/로 배포 완료" | - |

### 표준 요청 형식

```
[요청자]: {에이전트명}
[대상]: {에이전트명}
[유형]: PRD_REVIEW | IMPLEMENT | TEST | RELEASE_CHECK
[내용]: {상세 내용}
[첨부]: {관련 파일 경로}
```

### 표준 응답 형식

```
[응답자]: {에이전트명}
[판정]: APPROVED | NEEDS_REVISION | REJECTED | PASS | FAIL
[상세]: {판정 근거}
[액션]: {다음 단계 또는 수정 사항}
```

## 불변량 시스템 연동

### 각 에이전트별 불변량 사용

| 에이전트 | 불변량 사용 시점 | 사용 방식 |
|----------|----------------|----------|
| PLANNER | PRD 작성 시 | 해당 워크플로 불변량 참조하여 검증 기준 설계 |
| CRITIC | PRD 검증 시 | 불변량이 PRD에 제대로 반영되었는지 확인 |
| EXECUTOR | 검증기 구현 시 | 불변량 기반 ENHANCEMENT_RULES 정의 |
| TESTER | 테스트 시 | 불변량 위반 케이스 우선 테스트 |
| GATEKEEPER | 릴리즈 판정 시 | 불변량 체크리스트 최종 확인 |

### 불변량 흐름

```
workflow-invariants.json
         │
         ├──► PLANNER: PRD에 불변량 반영
         │
         ├──► CRITIC: 불변량 준수 여부 검증
         │
         ├──► EXECUTOR: validator.py에 불변량 구현
         │
         ├──► TESTER: 불변량 위반 테스트
         │
         └──► GATEKEEPER: 불변량 체크리스트 확인
```

## 규칙

### 필수 규칙

1. **PRD 없이 구현 금지** - PLANNER가 먼저 PRD 작성
2. **CRITIC 승인 없이 구현 금지** - APPROVED 받아야 EXECUTOR 진행
3. **테스트 없이 릴리즈 금지** - TESTER 통과 필수
4. **GATEKEEPER 승인 없이 배포 금지** - RELEASE 판정 필수

### 피드백 루프

- CRITIC → PLANNER: PRD 수정 요청
- TESTER → EXECUTOR: 버그 수정 요청
- GATEKEEPER → ALL: 릴리즈 거부 시 해당 단계로 되돌림

### 에스컬레이션

- 같은 단계에서 3회 연속 거부 → 상위 단계로 에스컬레이션
- PLANNER-CRITIC 교착 → 사용자 개입 요청
- EXECUTOR-TESTER 교착 → CRITIC 중재 요청

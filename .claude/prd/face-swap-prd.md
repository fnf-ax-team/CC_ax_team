# PRD: 얼굴 교체 (Face Swap)

> 작성일: 2026-02-19
> 상태: Draft

---

## 1. 개요

### 1.1 목적
기존 이미지의 **포즈/착장/배경을 100% 보존**하면서 **얼굴만 교체**하는 워크플로.
스튜디오 재촬영 없이 동일한 화보 구도에서 다른 모델의 얼굴로 빠르게 변환.

### 1.2 타겟 사용자
- 마케팅팀: 같은 화보 구도로 다른 모델/페르소나 버전 필요
- 브랜드팀: 캠페인 이미지의 모델 교체 (계약/시즌 변경)
- 이커머스팀: 동일 착장 기준으로 다양한 모델 얼굴 적용

### 1.3 핵심 가치
| 가치 | 설명 |
|------|------|
| 시간 절약 | 재촬영 없이 얼굴 교체 |
| 일관성 | 동일 포즈/착장/배경 유지 |
| 비용 절감 | 스튜디오/모델 재섭외 비용 감소 |

### 1.4 워크플로 카테고리
**스왑** - 얼굴만 변경, 포즈/착장/배경 유지

- [x] 얼굴 이미지 최대 5장 지원 (앙상블로 정확도 향상)

## 3. 입출력 정의

### 3.1 입력

| 입력 | 타입 | 필수 | 설명 |
|------|------|------|------|
| 소스 이미지 | Image | O | 포즈/착장/배경 보존 대상 (1장) |
| 얼굴 이미지 | Image[] | O | 교체할 얼굴 참조 (최대 5장) |

| 출력 | 타입 | 설명 |
|------|------|------|
| 생성 이미지 | Image[] | 얼굴 교체된 결과물 |
| 검증 결과 | dict | 점수, 등급, 이슈 |
| 분석 로그 | JSON | 소스/얼굴 분석 결과 |

### 3.3 출력 폴더
```
Fnf_studio_outputs/face_swap/{timestamp}/
├── result_01.png
```

## 5. 검증 기준

### 5.1 검증 기준 정의

| 기준 | 비중 | 설명 | Auto-Fail |
|------|------|------|-----------|
| face_identity | 40% | 얼굴 동일성 (참조 얼굴 vs 결과) | < 95 |
| pose_preservation | 25% | 포즈 유지 (관절 위치, 자세, 프레이밍) | < 95 |
| outfit_preservation | 20% | 착장 보존 (색상/로고/소재/실루엣) | < 95 |
| lighting_consistency | 10% | 조명 일관성 (광원 방향, 피부 하이라이트) | < 80 |
| edge_quality | 5% | 얼굴-배경/착장 경계 자연스러움 | < 80 |

**통과 조건: `total_score >= 95`**

### 5.2 등급 체계

| 등급 | 점수 | 판정 |
|------|------|------|
| S | 98+ | 바로 사용 |
| A | 95+ | 바로 사용 |
| B | 90+ | 확인 필요 |
| C | 80+ | 재생성 권장 |
| F | 80- | 재생성 필수 |

### 5.3 Auto-Fail 조건

| 조건 | 설명 |
|------|------|
| 다른 사람 얼굴 | face_identity < 95 |
| 포즈 변경됨 | pose_preservation < 95 |
| 착장 색상 불일치 | outfit color/logo match < 95 |
| 경계 부자연스러움 | edge_quality < 80 |
| 조명 불일치 | lighting 방향 역전 또는 극단적 불일치 |
| 누런 톤 | warm cast / 피부톤 황색화 |

### 5.4 MLB 브랜드 특화 검증

| 체크 항목 | 기준 |
|----------|------|
| NY/MLB 로고 위치 | 소스 이미지 동일 위치 유지 |
| 로고 색상 | 화이트/네이비/레드 정확 보존 |
| 로고 크기 | 소스 대비 ±5% |
| 착장 전체 실루엣 | 소스 착장 실루엣 100% 보존 |

### 5.5 재생성 로직

| 실패 기준 | 프롬프트 강화 방향 |
|----------|-------------------|
| face_identity 실패 | "EXACT same face features, skin tone, facial structure" 강조 |
| pose_preservation 실패 | "EXACT same pose, joint angles, body position unchanged" 강조 |
| outfit_preservation 실패 | "EXACT same outfit, colors, logos, fabric unchanged" 강조 |
| lighting_consistency 실패 | "Match lighting direction and intensity from source" 추가 |
| edge_quality 실패 | "Seamless blending at face boundary, natural skin transition" 추가 |

**재시도 설정:**
- 최대 재시도: 2회
- Temperature: 0.2 → 0.15 → 0.1

## 6. 기술 설계

### 6.1 모듈 구조

```

├── validator.py          # 검증 로직 (FaceSwapValidator)

```

### 6.2 기존 모듈 재사용

| 모듈 | 용도 |
|------|------|
| `core/outfit_analyzer.py` | 착장 분석 (보존 검증용) |
| `core/validators/base.py` | 검증기 베이스 클래스 |
| `core/api.py` | API 키 로테이션 |
| `core/config.py` | 모델 상수 |

### 6.3 API 사용

| 용도 | 모델 | Temperature |
|------|------|-------------|
| 소스 분석 | VISION_MODEL | 0.1 |
| 얼굴 분석 | VISION_MODEL | 0.1 |
| 이미지 생성 | IMAGE_MODEL | 0.2 |
| 검증 | VISION_MODEL | 0.1 |

### 6.4 이미지 전달 순서

```
1. 프롬프트 (텍스트) - 얼굴 교체 지시 + 소스 분석 설명
2. 소스 이미지 (첫 번째) - 포즈/착장/배경 보존 대상
3. 얼굴 이미지들 - 교체할 얼굴 참조 (최대 5장)
```

**핵심 원칙:**
- 소스 이미지: **API 직접 전달** (포즈/착장/배경 정확 보존)
- 얼굴 이미지: **API 직접 전달** (최대 5장, 앙상블 정확도)

### 6.5 face_images 처리 규칙

```python
# 최대 5장 제한
if len(face_images) > 5:
    face_images = face_images[:5]  # 첫 5개 사용, 사용자에게 알림

# 최소 1장 필수
if len(face_images) == 0:
    raise ValueError("얼굴 이미지는 최소 1장 필요")
```

## 7. 핵심 불변량 (Core Invariants)

### 7.1 보존 규칙

| 구분 | 요소 | 변경 허용 |
|------|------|----------|
| **must_preserve** | 포즈 (관절 위치) | 절대 불가 |
| **must_preserve** | 착장 (색상/로고/소재) | 절대 불가 |
| **must_preserve** | 배경 | 절대 불가 |
| **must_preserve** | 체형 | 절대 불가 |
| **must_change** | 얼굴 (동일 인물 교체) | 필수 변경 |
| **flexible** | 헤어스타일 경계 | 소스 포즈에 맞게 자연스럽게 |
| **flexible** | 목 조명 | 새 얼굴 피부톤에 맞게 조정 |

### 7.2 얼굴 경계 처리 규칙

얼굴이 소스 포즈/조명에 자연스럽게 블렌딩되어야 함:
- 목선 경계 → 자연스러운 피부 전환
- 헤어라인 → 소스 이미지 헤어와 자연스럽게 연결
- 귀 주변 → 착장 칼라/머플러 등과 충돌 없이
- 조명 방향 → 소스 조명과 동일하게 얼굴에 적용

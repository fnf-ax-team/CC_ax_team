# PRD: 트렌드 기반 콘텐츠 데이터 사이클 (Trend-Driven Content Data Cycle)

> 작성일: 2026-03-09
> 상태: Draft
> 기반: 회장님 발표 피드백 (2026-03-09)

---

## 1. 개요

### 1.1 목적

**트렌드 수집 → 검증 → 기획 지원 → 촬영/AI 생성 → 성과 추적**의 데이터 순환 시스템 구축.

현재 FNF Studio는 "입력 받은 이미지를 AI로 생성"하는 **도구(Tool)** 단계에 있다.
이를 **"데이터가 돌아가는 시스템(System)"**으로 진화시켜, 트렌드에 기반한 콘텐츠 기획부터 성과 측정까지 자동화한다.

### 1.2 배경 (회장님 피드백 요약)

| 피드백 | 핵심 질문 | 방향 |
|--------|----------|------|
| "트렌드 데이터는?" | 어떤 근거로 이 컨셉을 기획했는가 | AI 트렌드 크롤링 → 검증 |
| "기획 지원은?" | 트렌드+상품 데이터로 촬영 기획을 제안할 수 있는가 | AI 기획 어시스턴트 |
| "20/30 촬영 + 70/80 변형" | 핵심 촬영은 실제로, 나머지는 AI 변형 | 하이브리드 프로덕션 모델 |
| "히트율 공식" | Trend + Creative + Completeness | 성과 추적 → 피드백 루프 |
| "상품별 톤 전략" | 트렌드 따라가기 vs 브랜드 톤 유지 | 상품/카테고리별 차등 전략 |

### 1.3 핵심 가치

| 가치 | 설명 |
|------|------|
| 데이터 기반 기획 | 감이 아닌 데이터로 콘텐츠 방향 결정 |
| 비용 효율화 | 핵심 촬영 20-30% + AI 변형 70-80% |
| 속도 | 트렌드 감지 → 콘텐츠 배포 리드타임 단축 |
| 성과 측정 | 히트율 공식 기반 콘텐츠 성과 정량화 |
| 순환 학습 | 성과 데이터 → 다음 기획에 반영 |

### 1.4 현재 vs 목표

| 구분 | 현재 (Tool) | 목표 (System) |
|------|-------------|---------------|
| 트렌드 | 수동 (MD 감각) | AI 크롤링 + 자동 검증 |
| 기획 | 사람이 전부 기획 | AI 제안 + 사람 결정 |
| 촬영 | 100% 실촬영 or 100% AI | 20-30% 촬영 + 70-80% AI 변형 |
| 변형 | 수동 요청 | 자동 배리에이션 |
| 성과 | 정성적 판단 | 정량 공식 (Trend + Creative + Completeness) |
| 피드백 | 없음 | 성과 → 다음 기획 자동 반영 |

## 2. 시스템 아키텍처

### 2.1 전체 데이터 사이클

```
┌─────────────────────────────────────────────────────┐
│                 DATA CYCLE                           │
│                                                     │
│  [1. COLLECT]     AI 트렌드 크롤링                    │
│       │           (소셜/검색/경쟁사/글로벌)              │
│       ▼                                             │
│  [2. VERIFY]      내부 데이터 교차 검증                 │
│       │           (고객 구매/반응 데이터 대비)            │
│       ▼                                             │
│  [3. PLAN]        AI 기획 어시스턴트                    │
│       │           (트렌드 + 상품 → 촬영 기획안 제안)      │
│       ▼                                             │
│  [4. PRODUCE]     하이브리드 프로덕션                    │
│       │           (핵심 20-30% 촬영 + AI 70-80% 변형)   │
│       ▼                                             │
│  [5. DISTRIBUTE]  채널별 콘텐츠 배포                    │
│       │           (자동 리사이즈/포맷팅)                 │
│       ▼                                             │
│  [6. MEASURE]     성과 추적                           │
│       │           (Trend + Creative + Completeness)   │
│       │                                             │
│       └──────────────── [1] 로 순환 ──────────────────┘
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 2.2 모듈 구성

| 모듈 | 역할 | 우선순위 |
|------|------|---------|
| `trend_crawler` | AI 트렌드 크롤링 엔진 | P0 |
| `trend_verifier` | 내부 데이터 교차 검증 | P0 |
| `plan_assistant` | AI 기획 제안 엔진 | P1 |
| `hybrid_producer` | 하이브리드 프로덕션 오케스트레이터 | P1 |
| `variation_engine` | AI 변형 자동화 (배경/착장/포즈/컬러) | P1 |
| `channel_distributor` | 채널별 자동 배포 | P2 |
| `performance_tracker` | 성과 수집 + 히트율 계산 | P1 |
| `feedback_loop` | 성과 → 트렌드/기획 피드백 | P2 |

## 3. 모듈 상세

### 3.1 트렌드 크롤링 (trend_crawler) — P0

**AI로 패션 트렌드를 자동 수집하고 구조화한다.**

#### 크롤링 소스

| 소스 | 수집 대상 | 빈도 | 방법 |
|------|----------|------|------|
| Instagram | 해시태그, 인플루언서 피드, 릴스 트렌드 | 일 1회 | API + AI 이미지 분석 |
| TikTok | 바이럴 패션 콘텐츠, 해시태그 | 일 1회 | 공개 API + AI 영상 분석 |
| Pinterest | 인기 핀, 보드 트렌드 | 주 2회 | API |
| Google Trends | 검색 키워드 트렌드 | 일 1회 | API |
| Naver DataLab | 국내 검색 트렌드 | 일 1회 | API |
| 경쟁사 | Nike/Adidas/Zara/H&M 신상품 | 주 1회 | AI 크롤링 |
| 패션위크 | 런웨이/컬렉션 | 시즌별 | AI 이미지 분석 |
| 무신사/29CM | 인기 아이템, 리뷰 트렌드 | 일 1회 | AI 크롤링 |

#### 출력 형식

```json
{
  "trend_id": "TR-2026-0309-001",
  "detected_at": "2026-03-09T10:00:00",
  "category": "스트릿 캐주얼",
  "keywords": ["오버사이즈", "레트로 로고", "카고 팬츠"],
  "confidence": 0.85,
  "sources": [
    {"platform": "instagram", "posts": 12400, "growth_rate": "+23%/week"},
    {"platform": "tiktok", "views": "2.3M", "growth_rate": "+45%/week"}
  ],
  "visual_references": ["url1.jpg", "url2.jpg"],
  "ai_summary": "오버사이즈 실루엣과 레트로 로고 조합이 Z세대 중심으로 급부상...",
  "relevance_to_brands": {
    "MLB": {"fit": "HIGH", "reason": "빅로고 + 스트릿 정합"},
    "Discovery": {"fit": "LOW", "reason": "아웃도어와 거리 있음"}
  }
}
```

#### 핵심 기능

- [ ] 다중 소스 동시 크롤링 (병렬)
- [ ] AI 이미지 분석으로 비주얼 트렌드 추출 (색상/실루엣/소재/스타일)
- [ ] 트렌드 키워드 자동 클러스터링
- [ ] 브랜드별 관련성 자동 스코어링
- [ ] 주간/월간 트렌드 리포트 자동 생성
- [ ] 이상 신호 감지 (급부상/급하락)

### 3.2 트렌드 검증 (trend_verifier) — P0

**외부 트렌드를 내부 고객 데이터와 교차 검증하여 실행 가능성을 판단한다.**

"SNS에서 뜬다고 우리 고객이 사는 건 아니다."

#### 검증 로직

```
외부 트렌드 신호
       │
       ▼
┌─────────────────────────┐
│   내부 데이터 대비 검증     │
│                         │
│  1. 구매 데이터           │ ← 해당 카테고리/스타일 최근 매출 추이
│  2. 검색 데이터           │ ← 자사몰 내부 검색어 트렌드
│  3. 고객 프로필           │ ← 타겟 연령대/성별 매칭
│  4. 재고/MD 계획          │ ← 해당 상품 보유 여부, 시즌 적합성
│  5. 과거 성과             │ ← 유사 트렌드의 과거 히트율
│                         │
└─────────┬───────────────┘
          │
          ▼
   ┌──────────────┐
   │ 검증 결과      │
   │              │
   │ GO (실행)     │ → 기획 단계로
   │ WATCH (관찰)  │ → 모니터링 지속
   │ SKIP (패스)   │ → 기록만 남김
   └──────────────┘
```

#### 검증 스코어카드

| 항목 | 비중 | 평가 기준 |
|------|------|----------|
| 외부 신호 강도 | 20% | 소셜 성장률, 검색량 증가율 |
| 고객 매칭도 | 25% | 우리 고객 프로필과의 적합성 |
| 구매 데이터 연관성 | 25% | 유사 카테고리 최근 매출 추이 |
| 브랜드 적합성 | 15% | 브랜드 DNA와의 정합성 |
| 시즌 적시성 | 15% | 현재 시즌/MD계획과의 타이밍 |

#### 출력

```json
{
  "trend_id": "TR-2026-0309-001",
  "verification_result": "GO",
  "total_score": 82,
  "breakdown": {
    "external_signal": 90,
    "customer_match": 85,
    "purchase_correlation": 75,
    "brand_fit": 80,
    "season_timing": 78
  },
  "recommendation": "MLB 스트릿 캐주얼 라인에 즉시 반영 권장",
  "suggested_products": ["NY 빅로고 후디", "카고 조거팬츠", "레트로 캡"],
  "risk_factors": ["경쟁사 Nike SB가 유사 라인업 출시 예정"]
}
```

### 3.3 AI 기획 어시스턴트 (plan_assistant) — P1

**검증된 트렌드 + 상품 데이터 → 콘텐츠 기획안을 AI가 제안한다.**

사람은 제안을 검토/수정/승인하는 역할.

#### 기획 제안 프로세스

```
검증된 트렌드들 + 이번 시즌 상품 리스트 + 채널별 캘린더
                    │
                    ▼
           ┌─────────────────┐
           │ AI 기획 엔진      │
           │                 │
           │ 1. 트렌드-상품 매칭│
           │ 2. 컨셉 방향 제안  │
           │ 3. 촬영 구성 제안  │
           │ 4. 변형 계획 제안  │
           │ 5. 채널 배분 제안  │
           │ 6. 예산 산출      │
           └────────┬────────┘
                    │
                    ▼
           ┌─────────────────┐
           │ 기획안 (Draft)    │
           │                 │
           │ 담당자 검토       │
           │ → 수정 or 승인    │
           └─────────────────┘
```

### 3.4 하이브리드 프로덕션 (hybrid_producer) — P1

**핵심 촬영(20-30%)은 실제 촬영, 나머지(70-80%)는 AI 변형.**

#### 프로덕션 전략

```
                  전체 콘텐츠 필요량 (100장)
                         │
            ┌────────────┼────────────┐
            │            │            │
     핵심 촬영 (25장)  AI 변형 (75장)    │
     "히어로 컷"      "배리에이션"       │
            │            │            │
     실제 스튜디오    FNF Studio 자동    │
     고퀄리티 촬영         │            │
            │       ┌────┼────┐       │
            │       │    │    │       │
            │    배경변형 포즈변형 컬러변형  │
            │    (30장) (20장) (25장)   │
            │            │            │
            └────────────┼────────────┘
                         │
                    채널별 배포
```

#### 촬영 vs AI 변형 기준

| 구분 | 실제 촬영 (20-30%) | AI 변형 (70-80%) |
|------|-------------------|------------------|
| 용도 | 히어로컷, 키비주얼 | 채널별 변형, 배리에이션 |
| 품질 | 최고 품질 필요 | 히어로컷 기반 변형으로 충분 |
| 특성 | 브랜드 감성 원본 | 배경/포즈/컬러 변형 |
| 비용 | 높음 (촬영 비용) | 낮음 (API 비용) |

#### 변형 타입

| 변형 | 설명 | FNF Studio 워크플로 | 상태 |
|------|------|---------------------|------|
| 배경 변형 | 같은 인물, 다른 배경 | background-swap | ✅ 완료 |
| 포즈 변형 | 같은 인물/착장, 다른 포즈 | pose-change | 개발중 |
| 착장 변형 | 같은 인물, 다른 착장 | outfit-swap | 개발중 |
| 컬러 변형 | 같은 상품, 다른 컬러웨이 | (신규) color-variation | 미개발 |
| 사이즈 변형 | 같은 콘텐츠, 다른 비율 | 크롭/리사이즈 | ✅ 완료 |
| 채널 변형 | 같은 콘텐츠, 채널별 포맷 | channel-banner | ✅ 완료 |

### 3.5 상품별 톤 전략 — P1

**모든 상품에 같은 톤을 적용하지 않는다.**

회장님 피드백: "트렌드를 따라가는 상품도 있고, 브랜드 톤을 유지해야 하는 상품도 있다."

#### 톤 전략 매트릭스

| | 트렌드 민감도 HIGH | 트렌드 민감도 LOW |
|---|---|---|
| **브랜드 톤 강도 HIGH** | **A. 트렌드 어댑션** | **B. 브랜드 시그니처** |
| | 트렌드를 브랜드 필터로 해석 | 브랜드 톤 100% 유지 |
| | 예: MLB 시즌 콜라보 | 예: MLB 클래식 라인 |
| **브랜드 톤 강도 LOW** | **C. 트렌드 퍼스트** | **D. 뉴트럴** |
| | 트렌드 그대로 반영 | 기본 클린샷 |

### 3.6 성과 추적 (performance_tracker) — P1

**히트율 공식: Trend + Creative + Completeness**

#### 히트율 공식

```
HIT_RATE = (Trend_Score * 0.35) + (Creative_Score * 0.35) + (Completeness_Score * 0.30)
```

| 지표 | 비중 | 측정 방법 |
|------|------|----------|
| **Trend Score** | 35% | 트렌드 적중도 — 해당 트렌드 기반 콘텐츠의 실제 반응률 vs 예측 |
| **Creative Score** | 35% | 크리에이티브 품질 — 좋아요/저장/공유 비율, CTR, 체류시간 |
| **Completeness Score** | 30% | 완성도 — 채널 커버리지, 배리에이션 수, 일정 준수 |

#### Trend Score 세부

| 항목 | 측정 | 비중 |
|------|------|------|
| 트렌드 예측 적중 | 예측한 트렌드가 실제로 유효했는가 | 40% |
| 타이밍 적절성 | 트렌드 피크 대비 콘텐츠 배포 시점 | 30% |
| 경쟁사 대비 선점 | 경쟁사보다 먼저 반영했는가 | 30% |

#### Creative Score 세부

| 항목 | 측정 | 비중 |
|------|------|------|
| 인게이지먼트 | 좋아요 + 댓글 + 저장 + 공유 / 노출 | 40% |
| CTR | 클릭률 (광고/배너 기준) | 30% |
| 전환 기여 | 콘텐츠 → 상품페이지 → 구매 전환 | 30% |

#### Completeness Score 세부

| 항목 | 측정 | 비중 |
|------|------|------|
| 채널 커버리지 | 계획 채널 대비 실제 배포 채널 비율 | 35% |
| 변형 다양성 | 계획 변형 수 대비 실제 생산 수 | 35% |
| 일정 준수 | 계획 대비 실제 배포일 차이 | 30% |

#### 성과 대시보드 출력


### 3.7 피드백 루프 (feedback_loop) — P2

**성과 데이터가 다음 기획 사이클에 자동으로 반영된다.**

```
성과 데이터 수집
      │
      ▼
┌─────────────────────┐
│ 학습 포인트 추출       │
│                     │
│ - 어떤 트렌드가 맞았나  │ → trend_crawler 가중치 조정
│ - 어떤 크리에이티브가   │ → plan_assistant 선호 패턴 학습
│   잘 먹혔나           │
│ - AI vs 실촬영 차이는  │ → hybrid_producer 비율 최적화
│ - 어떤 변형이 효과적   │ → variation_engine 우선순위 조정
│ - 채널별 성과 차이는   │ → channel_distributor 배분 최적화
│                     │
└─────────────────────┘
      │
      ▼
다음 사이클 기획에 자동 반영
```

# 챗봇 QA 테스트 스킬

사용자가 입력한 질문 리스트에 대해 챗봇 API를 호출하고, 결과를 노션 DB에 저장하는 스킬입니다.

## 사용 방법

1. 사용자가 질문 리스트를 입력합니다 (콤마 또는 줄바꿈으로 구분)
2. 스킬이 각 질문에 대해 챗봇 API를 호출합니다
3. 결과가 새로운 노션 DB에 저장됩니다

## 입력 형식

**콤마로 구분:**
```
SA시스템 정산 어떻게해?, 연차 신청 방법, 법인카드 한도 증액
```

**줄바꿈으로 구분:**
```
SA시스템 정산 어떻게해?
연차 신청 방법
법인카드 한도 증액
```

## 노션 DB 스키마

각 실행마다 새로운 노션 DB가 생성되며, 다음 속성을 가집니다:

| 속성명 | 타입 | 설명 |
|--------|------|------|
| 질문 | Title | 사용자 질문 (페이지 제목) |
| 부서 | Text | 질문이 분류된 담당 부서 |
| 답변 | Text | 챗봇 API 응답 |
| 결과 | Select | QA 결과 (Pass/Fail/검토필요) - 사용자가 직접 선택 |

## 부서 분류 키워드

질문 내용을 기반으로 자동으로 부서를 분류합니다:

- **총무팀**: 회의실, 주차, 명함, 리조트, 동호회, 레벨업, 야근택시, 건강검진, 식당, 헬스장
- **인사팀**: 연차, 휴가, 급여, 수당, 승진, 평가, 퇴직, 재직증명서, 출퇴근
- **회계팀**: SA, 정산, 전표, 품의, 출장, 세금계산서, 비용, 회식비
- **자금팀**: 법인카드, 카드한도, 한도증액
- **법무팀**: ELAW, 계약서, 인감, 상표
- **정보보안팀**: WiFi, VPN, VDI, 비밀번호, 보안, NAC
- **IT팀**: SAP, MERP, SERP, 시스템, 팀즈, 노트북
- **커뮤니케이션팀**: 브랜드소개, 홍보, DT2.0, 회장님

## 실행 코드

스킬 실행 시 `qa_test_runner.py` 파일을 사용합니다.

```python
# 사용 예시
python qa_test_runner.py "질문1, 질문2, 질문3"
```

## 노션 설정

노션 API 키와 부모 페이지 ID가 필요합니다:
- NOTION_API_KEY: 노션 통합 API 키
- PARENT_PAGE_ID: DB가 생성될 부모 페이지 ID

## 주의사항

- 챗봇 API 호출은 병렬로 처리됩니다 (max_workers=5)
- 각 질문당 타임아웃은 120초입니다
- API 호출 실패 시 답변에 "API 호출 실패" 메시지가 저장됩니다

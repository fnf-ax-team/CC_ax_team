## 4. 워크플로 설계

### 4.1 단계별 흐름

```
1. 입력 수집      → 대화형 질문 (AskUserQuestion)
2. VLM 분석       → 착장/포즈/배경 분석
3. 프롬프트 조립   → build_prompt()
4. 이미지 생성    → generate()
5. 검증           → validator.validate()
6. 재생성 (옵션)  → 실패 시 프롬프트 강화 후 재시도
7. 출력           → 저장 + 결과 반환
```

### 4.2 대화형 질문 설계

**1단계: 필수 입력**
| 질문 | 옵션들 | 기본값 |
|------|--------|--------|
| 얼굴 폴더 경로 | 경로 입력 | - |
| 착장 폴더 경로 | 경로 입력 | - |

**2단계: 옵션 선택**
| 질문 | 옵션들 | 기본값 |
|------|--------|--------|
| 비율 | 1:1, 3:4, 9:16, ... | 3:4 |
| 수량 | 1, 3, 5, 10 | 3 |
| 화질 | 1K, 2K, 4K | 2K |

## 6. 기술 설계

### 6.1 모듈 구조

```
core/{워크플로명}/
├── __init__.py           # 통합 진입점
├── analyzer.py           # VLM 분석
├── prompt_builder.py     # 프롬프트 조립
├── generator.py          # 생성 + 검증
├── validator.py          # 검증 로직
└── templates.py          # 프롬프트 템플릿
```

### 6.2 API 사용

| 용도 | 모델 | Temperature |
|------|------|-------------|
| VLM 분석 | gemini-3-flash-preview | 0.1 |
| 이미지 생성 | gemini-3-pro-image-preview | 0.25 |
| 검증 | gemini-3-flash-preview | 0.1 |

### 6.3 스킬 파일 구조

```
.claude/skills/{워크플로명}/
├── SKILL.md              # 스킬 정의
├── {name}-cheatsheet.md  # 프롬프트 치트시트
└── validator.py          # 스킬 전용 검증기 (옵션)
```

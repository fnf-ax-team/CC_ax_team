## 핵심 원칙 (v2.0)

```
+--------------------------------------------------------------------+
|  이미지가 말한다. 텍스트는 역할만 설명한다.                            |
+--------------------------------------------------------------------+
```

### v1.0 vs v2.0 차이

| 항목 | v1.0 (텍스트 프롬프트) | v2.0 (이미지 레퍼런스) |
|------|----------------------|----------------------|
| 포즈 | JSON 스키마 상세 기술 | **포즈 레퍼런스 이미지** |
| 표정 | 눈/입/바이브 텍스트 | **표정 레퍼런스 이미지** |
| 배경 | 장소/분위기 텍스트 | **배경 레퍼런스 이미지** |
| 착장 | 텍스트 + 이미지 | **착장 이미지만** |
| 프롬프트 길이 | 수백 단어 | **역할 설명 50단어** |

**왜 이미지 레퍼런스인가?**
- 텍스트로 포즈/표정 정확 묘사 어려움
- 이미지가 훨씬 정확한 정보 전달
- 프롬프트 간결 → 모델 혼동 감소
- 프리셋 이미지로 일관된 결과

## 1. 이미지 역할 (API 전송 순서)

### 표준 프롬프트 (v2.0)

```
[IMAGE ROLES]

IMAGE 1: POSE/EXPRESSION REFERENCE
- Copy EXACT pose, camera angle, framing
- Copy EXACT expression (mouth, eyes, vibe)
- Ignore face/outfit from this image

IMAGE 2-4: FACE REFERENCE (multi-angle)
- Use ONLY this person's face
- Maintain face identity across all angles
- Weight: 40%

IMAGE 5+: OUTFIT REFERENCE
- Wear EXACTLY these clothes
- Match colors, logos, details precisely

IMAGE LAST: BACKGROUND REFERENCE (if provided)
- Use this environment/setting
- Ignore any people in background image

[OUTPUT]
- Natural influencer photo
- Authentic, not overly polished
- Realistic skin texture
```

### API 전송 순서

```python
contents = [
    prompt_text,          # 역할 설명 (위 텍스트)
    pose_image,           # IMAGE 1: 포즈/표정 레퍼런스
    *face_images,         # IMAGE 2-4: 얼굴 (3-5장)
    *outfit_images,       # IMAGE 5+: 착장
    background_image,     # IMAGE LAST: 배경 (선택)
]
```

## 2. 캐릭터 설정

### 폴더 구조

```
db/ai_influencer/{캐릭터명}/
├── profile.json           <- 캐릭터 프로필
├── face/                  <- 기준 얼굴 이미지들 (3-5장)
│   ├── front.jpg          <- 정면 (필수)
│   ├── side.jpg           <- 측면 (권장)
│   └── smile.jpg          <- 미소 (권장)
└── style_guide.md         <- 스타일 가이드 (선택)
```

### profile.json 형식

```json
{
  "name": "루나",
  "name_en": "Luna",
  "gender": "여성",
  "age": "20대 초반",
  "ethnicity": "한국인",
  "face_features": {
    "face_shape": "계란형, 갸름한 턱선",
    "eyes": "크고 또렷한 쌍커풀 눈, 긴 속눈썹",
    "nose": "높고 오똑한 콧대",
    "lips": "도톰한 입술",
    "skin": "맑고 투명한 피부톤"
  },
  "style": {
    "brand_affinity": ["MLB", "스트릿", "캐주얼"],
    "preferred_colors": ["블랙", "화이트", "데님블루"],
    "makeup_style": "natural"
  },
  "personality": "밝고 친근한, 일상을 공유하는 인플루언서"
}
```

### 다각도 얼굴 이미지 (CRITICAL)

| 이미지 | 역할 | 필수 |
|--------|------|------|
| `front.jpg` | 정면 얼굴 | O |
| `side.jpg` | 측면 (3/4 앵글) | 권장 |
| `smile.jpg` | 미소/표정 변화 | 권장 |
| `angle1.jpg` | 추가 앵글 | 선택 |
| `angle2.jpg` | 추가 앵글 | 선택 |

**왜 3-5장인가?**
- 단일 이미지로는 얼굴 동일성 유지 어려움
- 다양한 각도로 얼굴 특징 학습
- 조명/각도 변화에도 일관된 생성

## 3. 프리셋 이미지 연동

### 프리셋 폴더 구조

```
db/presets/
├── pose/                    # 포즈 레퍼런스 이미지
│   ├── 전신_01.jpg ~ 전신_21.jpg
│   ├── 상반신_01.jpg ~ 상반신_21.jpg
│   ├── 앉기_01.jpg ~ 앉기_21.jpg
│   └── 거울셀피_01.jpg ~ 거울셀피_12.jpg
├── expression/              # 표정 레퍼런스 이미지
│   ├── 시크_01.jpg ~ 시크_05.jpg
│   └── 러블리_01.jpg ~ 러블리_05.jpg
└── background/              # 배경 레퍼런스 이미지
    ├── 핫플카페_01.jpg ~ 핫플카페_21.jpg
    ├── 그래피티_01.jpg ~ 그래피티_15.jpg
    └── ...
```

### 포즈 프리셋 (75개)

| 카테고리 | ID 범위 | 개수 | 특징 |
|----------|---------|------|------|
| **전신** | `전신_01` ~ `전신_21` | 21 | 로우앵글, 다리 길어보이게, 역동적 |
| **상반신** | `상반신_01` ~ `상반신_21` | 21 | 얼굴 가까이, 팔/손 포즈 |
| **앉기** | `앉기_01` ~ `앉기_21` | 21 | 쪼그려, 계단, 바닥, 벤치 |
| **거울셀피** | `거울셀피_01` ~ `거울셀피_12` | 12 | 거울 앞, 플래시, 인스타 스타일 |

### 표정 프리셋 (10개)

| 카테고리 | ID 범위 | 특징 |
|----------|---------|------|
| **시크** | `시크_01` ~ `시크_05` | 쿨, 도도, 시크, 셀카 앵글 |
| **러블리** | `러블리_01` ~ `러블리_05` | 부드러움, 몽환적, 귀여움 |

### 배경 프리셋 (96개)

| 카테고리 | 개수 | 스타일 |
|----------|------|--------|
| **핫플카페** | 21 | 유럽풍, 모던, 레트로 |
| **그래피티** | 15 | 스트릿 아트, 힙한 느낌 |
| **철문** | 10 | 금속 셔터, 인더스트리얼 |
| **기타문** | 10 | 유럽 빈티지 문 |
| **해외스트릿** | 10 | 홍콩, 파리, 런던, 뉴욕 |
| **힙라이프** | 10 | 아트북숍, 편의점, 레코드샵 |
| **지하철** | 10 | 통로, 플랫폼, 객차 |
| **엘레베이터** | 5 | 거울 엘리베이터 |
| **횡단보도** | 5 | 유럽풍 교차로 |

### 프리셋 ID → 이미지 경로

```python
def get_preset_image_path(category: str, preset_id: str) -> Path:
    """프리셋 ID로 이미지 경로 반환"""
    base = Path("db/presets")
    return base / category / f"{preset_id}.jpg"

# 사용 예시
pose_img = get_preset_image_path("pose", "전신_05")
# -> db/presets/pose/전신_05.jpg

expr_img = get_preset_image_path("expression", "시크_02")
# -> db/presets/expression/시크_02.jpg
```

## 4. 호환 규칙

### 포즈 ↔ 배경 호환

| 포즈 카테고리 | 호환 배경 | 비호환 |
|--------------|----------|--------|
| **전신** | 모든 배경 | - |
| **상반신** | 대부분 | 횡단보도 |
| **앉기** | 카페, 계단, 힙라이프 | 횡단보도, 철문 |
| **거울셀피** | 엘레베이터만 | **야외 전체** |

### 금지 조합

| 조합 | 이유 |
|------|------|
| 거울셀피 + 야외 배경 | 거울이 야외에 없음 |
| 앉기 + 횡단보도 | 상황 부자연스러움 |
| 전신 포즈 + 얼굴 클로즈업 | 물리적 모순 |

### 포즈+표정 통합 vs 분리

**통합 사용 (권장):**
- 포즈 레퍼런스 이미지의 표정까지 그대로 사용
- 표정 프리셋 별도 지정 안함

**분리 사용:**
- 포즈 레퍼런스와 다른 표정 원할 때
- 별도 표정 프리셋 이미지 전송

## 5. 검증 기준

### 5개 항목 (얼굴 동일성 40%)

| 항목 | 영문 | 비중 | 설명 |
|------|------|------|------|
| **얼굴 일관성** | face_consistency | **40%** | 캐릭터 얼굴과 동일한가 |
| 리얼리즘 | realism | 25% | 실제 사진처럼 보이는가 |
| 시나리오 적합성 | scenario_fit | 15% | 장소/상황이 자연스러운가 |
| 피부 상태 | skin_condition | 10% | 피부 질감이 자연스러운가 |
| 역검증 | anti_polish_factor | 10% | 너무 완벽하지 않은가 |

### Auto-Fail 조건

- **얼굴 다른 사람** (가장 중요!)
- 손가락 6개+ / 기형적 손가락
- AI 특유 플라스틱 피부

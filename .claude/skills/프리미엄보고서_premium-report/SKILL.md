범용적이지 않은, 독창적이고 발표 수준의 단일 HTML 보고서를 생성하는 스킬. "AI가 만든 티"가 나는 뻔한 미감을 철저히 배제한다. 모든 보고서는 독립적인 `.html` 파일 — CSS와 JS는 인라인, 폰트 CDN만 외부 의존성으로 허용.

## 콘텐츠 유형

사용자가 만드는 보고서는 주로 다음 유형이다. 유형에 맞는 콘텐츠 구성을 잡아준다.

| 유형 | 청중 | 핵심 구성 |
|------|------|----------|
| 사내 커뮤니케이션 | 임직원 전체 | 인사말 + 핵심 메시지 + 브랜드별 현황 + 비전 |
| 경영진 보고 | 회장/임원 | 핵심 KPI + 성과 요약 + 전략 방향 + 로드맵 |
| 팀 성과 보고 | 팀/부서 | 활동 현황 + 수치 실적 + 사례 쇼케이스 + 다음 계획 |
| 기술 소개 | 사내/외부 | 기술 개요 + 파이프라인 + Before/After + 활용 사례 |
| 브랜드 리뷰 | 내부 관계자 | 브랜드별 성과 + 주요 컨텐츠 + 시장 반응 + 개선점 |
| 프로젝트 제안 | 의사결정자 | 배경 + 목표 + 접근법 + 기대효과 + 일정 |

유형이 명확하지 않으면 사용자에게 질문한다.

## 디자인 사고

코딩 전에 맥락을 파악하고 대담한 미적 방향을 선택한다:
- 목적: 이 보고서는 무엇에 관한 것인가? 청중은 누구인가?
- 톤: 극단을 택한다: 극도의 미니멀, 맥시멀리스트, 레트로-퓨처리스틱, 유기적/자연적, 럭셔리/정제된, 에디토리얼/매거진, 브루탈리스트/날것, 아르데코/기하학적, 시네마틱/느와르, 스위스-타이포그래피, 일본-젠, 소프트/파스텔, 산업적/실용적 등. 보고서 맥락에 진정성 있는 방향을 설계한다.
- 차별화: 무엇이 이 보고서를 잊을 수 없게 만드는가?

핵심: 명확한 컨셉 방향을 정하고 정밀하게 실행한다. **매 보고서가 이전과 달라야 한다.** 같은 디자인으로 수렴하지 않는다.

## 보고서 미적 가이드라인

집중해야 할 것:
- 타이포그래피: 아름답고 독특한 폰트를 선택한다. Arial이나 Inter 같은 범용 폰트를 피한다. 한국어 보고서에는 양질의 한국어 폰트(Pretendard, SUIT, Noto Sans KR 등)를 추가한다.
- 색상 & 테마: CSS 변수(`:root`)로 응집된 미감을 구현한다. 지배적인 색상에 날카로운 포인트. 항상 다크/라이트 테마 토글을 `html.theme-light` 클래스로 제공한다.
- 모션: 스크롤 트리거 페이드업(`IntersectionObserver`)을 기본으로. 잘 연출된 페이지 로드가 흩어진 효과보다 낫다.
- 공간 구성: 넉넉한 섹션 간격(`clamp()`). 예상치 못한 레이아웃. 그리드를 깨는 순간들.
- 배경: 단색 대신 분위기와 깊이. 그라디언트 메시, 노이즈 텍스처, 레이어드 투명도.

금지: Inter/Roboto/Arial 단독, 보라 그라디언트+흰 배경, 예측 가능한 레이아웃, 맥락 없는 틀에 박힌 디자인.

## 필수 보고서 구조

1. **스크롤 진행 바** — 상단 고정, 읽기 진행도 표시
2. **고정 내비게이션** — 스크롤 시 `backdrop-filter: blur()`, `IntersectionObserver`로 섹션 추적, 테마 토글
3. **히어로 섹션** — `100vh` 풀스크린, 하단 정렬 콘텐츠, 그라디언트/이미지 배경
4. **목차** — 그리드 또는 리스트, 3~6항목, 클릭 네비게이션
5. **콘텐츠 섹션** — 각각 아이브로우 레이블 + 제목 + 부제목 패턴
6. **페이드업 애니메이션** — `.fade-up` + `.visible` via `IntersectionObserver`, 시차 딜레이
7. **맨 위로 버튼** — 플로팅, 스크롤 시 나타남

## 출력 형식

항상 단일 `.html` 파일로 생성:

```html
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>보고서 제목</title>
  <!-- 폰트 CDN만 허용 -->
  <style>/* 모든 CSS 인라인 */</style>
</head>
<body>
  <!-- 컴포넌트 조합 -->
  <script>/* 모든 JS 인라인 */</script>
</body>
</html>
```

생성 후 "파일을 열어드릴까요?" 질문하고 승인 시 OS 기본 브라우저로 오픈.

## 디자인 DO / DON'T

**DO:** CSS 변수로 디자인 토큰, `clamp()`으로 반응형 타이포그래피, 3단계 텍스트 위계, 은은한 보더, 넉넉한 여백, `loading="lazy"`, `@media print` 지원.

**DON'T:** 보라 그라디언트+흰 배경, Inter/Roboto/Arial 단독, 단색 팔레트, 과한 `box-shadow`, 16px 초과 `border-radius`, 빈 이미지 플레이스홀더, 3초 넘는 애니메이션, 아이콘 남용.

매 보고서마다 다른 선택을 한다. 라이트/다크, 다른 폰트, 다른 미감. **절대 공통 선택으로 수렴하지 않는다.**

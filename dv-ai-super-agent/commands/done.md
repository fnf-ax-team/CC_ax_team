# 하루 마무리

작업 완료 후 문서 동기화, 커밋, 푸시를 한 번에 진행합니다.

## 실행 순서

### Step 0: input 폴더 파일 체크 (필수!)
```bash
git status input/ --porcelain
```
- **untracked 파일(??)**이 있으면 `git add input/` 실행
- input 폴더의 모든 파일(csv, json, md 등)은 반드시 GitHub에 포함되어야 함
- ⚠️ 이 체크를 통과해야 다음 단계 진행

### Step 1: /sync - 문서 동기화
- CLAUDE.md 세션 진행 현황 업데이트
- 최근 작업 내용 반영

### Step 2: /commit - Git 커밋
- 변경사항 확인 (git status, git diff --stat)
- 커밋 메시지 자동 생성
- 스테이징 및 커밋 (input 폴더 포함)

### Step 3: /push - GitHub 푸시
- 로컬 커밋을 원격 저장소에 업로드
- 결과 확인

## 출력 형식

```
========================================
🔍 Step 0: input 폴더 체크
========================================
[✅ 모든 파일 추적 중 / ⚠️ N개 파일 추가됨]

========================================
🔄 Step 1/3: 문서 동기화
========================================
[sync 결과]

========================================
📝 Step 2/3: Git 커밋
========================================
[commit 결과]

========================================
🚀 Step 3/3: GitHub 푸시
========================================
[push 결과]

========================================
✅ 완료!
========================================
- 커밋: [해시]
- GitHub: https://github.com/[org]/[repo]
```

## 주의사항
- **input 폴더 untracked 파일은 반드시 추가** (프로젝트 필수 데이터)
- 각 단계에서 오류 발생 시 중단하고 사용자에게 알림
- 변경사항이 없으면 커밋/푸시 단계 스킵
